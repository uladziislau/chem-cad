# Архитектура ООП-движка: Законы химии в коде

## Философия движка
Наш химический движок строится на фундаментальном уважении к труду ученых. Мы не используем "черные ящики" ИИ для угадывания там, где наука уже дала точные математические формулы. 
Каждый объект в системе подчиняется открытым законам физики и химии. ООП (Объектно-Ориентированное Программирование) выступает идеальным языком для описания этих законов.

## 1. Паттерн "Компоновщик" (Composite) для Веществ
В химии смесь ведет себя как новое вещество со своими свойствами. В ООП это идеально описывается паттерном Composite.

```typescript
// Базовый интерфейс для ЛЮБОГО химического объекта
interface IChemicalEntity {
    getMass(): number;
    getVaporPressure(temperature: number): number;
}
```

## 2. Инкапсуляция законов природы

### Закон Антуана (Давление насыщенного пара)
Ученый Луи Шарль Антуан в 1888 году вывел формулу давления пара. В нашем коде чистое вещество инкапсулирует этот закон:

```typescript
class PureSubstance implements IChemicalEntity {
    constructor(
        public name: string, 
        public molarMass: number,
        private antoineA: number,
        private antoineB: number,
        private antoineC: number
    ) {}

    getMass(): number { return this.molarMass; }

    // Реализация Закона Антуана: log10(P) = A - (B / (T + C))
    getVaporPressure(T_Celsius: number): number {
        const logP = this.antoineA - (this.antoineB / (T_Celsius + this.antoineC));
        return Math.pow(10, logP);
    }
}
```

### Закон Рауля (Свойства смесей)
Ученый Франсуа Мари Рауль в 1887 году доказал, что парциальное давление компонента в идеальной смеси равно его давлению умноженному на мольную долю.
Объект `Mixture` реализует этот закон через полиморфизм:

```typescript
class Mixture implements IChemicalEntity {
    private components: { entity: IChemicalEntity, moleFraction: number }[] = [];

    getMass(): number {
        // Аддитивность масс
        return this.components.reduce((sum, c) => sum + (c.entity.getMass() * c.moleFraction), 0);
    }

    // Реализация Закона Рауля
    getVaporPressure(T_Celsius: number): number {
        return this.components.reduce((totalPressure, c) => {
            // Полиморфный вызов: нам неважно, чистое это вещество или другая смесь!
            const purePressure = c.entity.getVaporPressure(T_Celsius);
            return totalPressure + (purePressure * c.moleFraction);
        }, 0);
    }
}
```

## 3. Эмерджентность в Рантайме (Runtime)
Благодаря такой архитектуре, на этапе выполнения (в рантайме) происходит магия:
1. Пользователь создает смесь из Воды и Этанола.
2. Пользователь помещает эту смесь внутрь другой смеси (например, добавляет водно-спиртовой раствор в крем).
3. Когда UI запрашивает `getVaporPressure(25)` у финального крема, система рекурсивно спускается по дереву объектов.
4. Каждый объект применяет свой закон (Рауля, Антуана и т.д.) и возвращает точный научный результат.

## 4. Решатели (Solvers)
Законы позволяют нам строить "Решатели". Например, чтобы найти **Температуру кипения** смеси, нам не нужна новая формула. 
Температура кипения — это момент, когда `getVaporPressure(T) == Атмосферному давлению`.
Наш `ThermodynamicsSolver` просто использует метод бисекции (поиск корня), подставляя разные `T` в метод `getVaporPressure`, пока не найдет точку кипения. 

**Итог:** Запрограммировав всего 2 закона (Антуана и Рауля), мы автоматически получаем возможность рассчитывать температуру кипения для бесконечного множества комбинаций смесей. Это и есть мощь ООП, помноженная на науку.
