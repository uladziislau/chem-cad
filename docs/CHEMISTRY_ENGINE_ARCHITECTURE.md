# Архитектура Профессионального Химического ООП-Движка (ChemCore)

## Концепция
Для создания симулятора профессионального уровня необходимо отказаться от школьной модели "шариков и палочек" (где валентность — это просто число) и перейти к строгим физико-химическим моделям, основанным на **Теории графов**, **Квантовой механике** и **Термодинамике**.

## Архитектурные слои (Layers)

### Концепция: Распределенная "Система Систем" (Multiscale Architecture)
Современный химический движок не может быть монолитом. Физика работает по-разному на разных масштабах. Наша архитектура строится как набор независимых микросервисов (Акторов), общающихся через асинхронную шину событий (Event Bus). Это позволяет выносить тяжелые расчеты в WebWorkers и WebGPU.

### 1. Квантовый узел (Quantum Node) - *Масштаб: Электроны*
Базовый уровень, описывающий состояние электронов.
*   **Задача**: Расчет точных энергий, гибридизации, распределения зарядов (ab initio).
*   **`IOrbital`**: Описывает атомные (s, p, d, f) и молекулярные (σ, π) орбитали.
*   **`IElectron`**: Имеет `spin` (+1/2 или -1/2).

### 2. Топологический узел (Topology Node) - *Масштаб: Графы*
Молекула профессионального уровня — это взвешенный неориентированный граф.
*   **Задача**: Хранение структуры, парсинг SMILES, поиск изомеров и циклов.
*   **`AtomNode` (Узел)**: Хранит формальный заряд, гибридизацию.
*   **`BondEdge` (Ребро)**: Хранит порядок связи, энергию, длину.

### 3. Узел Молекулярной Динамики и Рендеринга (MD & VMD Node) - *Масштаб: Атомы*
*   **Задача**: Симуляция нековалентных взаимодействий и 3D-визуализация в реальном времени.
*   **Движок (WebGPU + Three.js)**: Используется архитектура **Zero-Copy**. 
    *   **Compute**: Вычислительные шейдеры (WGSL) параллельно рассчитывают задачу N-тел (Леннард-Джонс) и обновляют Storage Buffers прямо в VRAM видеокарты.
    *   **Render**: `WebGPURenderer` из Three.js использует паттерн `InstancedMesh` для отрисовки десятков тысяч 3D-сфер, читая координаты напрямую из тех же Storage Buffers, минуя CPU и шину данных.

### 4. Термодинамический и Кинетический узел (Reactor Node) - *Масштаб: Ансамбли*
Управляет энергией, фазовыми состояниями и скоростью реакций в макро-объемах.
*   **Задача**: Расчет энтальпии (Закон Гесса), симуляция столкновений (Монте-Карло).
*   **`IEnvironment` (Колба/Реактор)**: `temperature`, `pressure`, `solvent`.
*   **`IReactionMechanism`**: Паттерн Стратегия для пошагового описания реакции.

### 5. AI-Оркестратор (Cheminformatics Node) - *Масштаб: Идеи*
*   **Задача**: Генерация гипотез (SMILES), анализ результатов симуляции, управление пайплайном Drug Discovery.

## Применяемые Паттерны Проектирования (Design Patterns)

1.  **Builder (Строитель)**: `MoleculeBuilder` для сборки сложных молекул (например, белков или полимеров) шаг за шагом, рассчитывая минимизацию энергии (VSEPR).
2.  **Strategy (Стратегия)**: Для расчета свойств. Например, `IEnergyCalculator` может иметь реализации `MolecularMechanicsCalculator` (быстро) или `QuantumMechanicsCalculator` (медленно, но точно).
3.  **State (Состояние)**: Для агрегатных состояний. Поведение молекулы воды в `IceState` отличается от `SteamState`.
4.  **Observer (Наблюдатель)**: Среда (`Environment`) уведомляет все молекулы об изменении температуры, что может триггернуть распад или изменение состояния.
5.  **Visitor (Посетитель)**: Для обхода графа молекулы (например, чтобы найти все гидроксильные группы или рассчитать общую массу, не меняя структуру графа).

## План разработки (Roadmap)

- [x] **Фаза 1: Ядро Графов**. Переписать `Molecule` как математический граф. Реализовать алгоритмы поиска (например, поиск циклов для ароматичности).
- [x] **Фаза 2: Квантовая механика**. Внедрить орбитали и гибридизацию вместо жесткой `maxValence`.
- [x] **Фаза 3: Термодинамика и Эмерджентность**. Внедрить Закон Гесса, Потенциал Леннард-Джонса и Live-колбу.
- [ ] **Фаза 4: Визуальная Молекулярная Динамика (VMD)**. Создание 2D/3D Canvas-движка для рендеринга физики в реальном времени.
- [ ] **Фаза 5: Хемоинформатика (Cheminformatics)**. Интеграция парсера SMILES (например, `CC(=O)OC1=CC=CC=C1C(=O)O` для Аспирина) для алгоритмической генерации графов.
- [ ] **Фаза 6: WebWorkers & GPU Acceleration**. Вынос тяжелых расчетов MD (Molecular Dynamics) в фоновые потоки для симуляции тысяч атомов.
- [ ] **Фаза 7: AI-Driven Discovery**. Подключение LLM для предиктивного синтеза и поиска новых молекул под заданные свойства (Drug Discovery).
