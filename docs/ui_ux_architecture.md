# Архитектура пользовательского интерфейса (UI/UX)

## Концепция: "Химическая IDE" (Integrated Development Environment)
Интерфейс должен отходить от концепции "веб-странички" и приближаться к профессиональным инструментам проектирования (как Figma, VS Code, AutoCAD или Blender). Пользователь должен чувствовать, что он находится за пультом управления высокотехнологичной лабораторией.

**Визуальный стиль:**
- **Тема:** Темная (Dark Mode) — `slate-950` фон, `slate-200` текст. Снижает нагрузку на глаза при долгой работе.
- **Акценты:** `indigo` (наука/интеллект), `emerald` (успех/стабильность), `rose` (ошибки/опасность).
- **Типографика:** Рубленые шрифты (Inter) для интерфейса и моноширинные (JetBrains Mono) для данных, формул и логов.

## Глобальный Layout (Каркас приложения)
Приложение строится по классической схеме профессиональных редакторов:

1. **Top Bar (Верхняя панель):**
   - Логотип и переключение главных модулей (Склад, Рецептуры, Синтез, Анализ).
   - Глобальные статусы (Индикатор работы WebGPU, статус локальной БД).

2. **Left Sidebar (Обозреватель/Палитра):**
   - Контекстное меню для текущего модуля.
   - В "Рецептурах" — список доступных ингредиентов.
   - В "Синтезе" — палитра типовых реакций.

3. **Main Canvas (Главная рабочая область):**
   - Центральная часть, где происходит магия.
   - 3D-вьюер молекулы, таблица рецептуры или Node-based (узловой) граф химической реакции.

4. **Right Sidebar (Инспектор свойств):**
   - Детальная информация о выделенном объекте.
   - Если выделена молекула: её LogP, TPSA, масса.
   - Если выделена смесь: её pH, прогноз стабильности, цена.

5. **Bottom Panel (Терминал/Журнал):**
   - Сворачиваемая панель внизу.
   - Вывод логов ChemCore, ошибок GPU, предупреждений о токсичности.

## Главные разделы (Роутинг)

### 1. `/inventory` (Склад Сырья)
- **Суть:** База данных прекурсоров. Что у нас есть в наличии или что можно легко купить.
- **UI:** Таблица с поиском и фильтрами (ПАВы, Растворители, Соли, Кислоты). Каждая карточка сырья содержит его химическую структуру и цену.

### 2. `/formulator` (Проектирование Рецептур)
- **Суть:** Создание смесей (косметика, бытовая химия, удобрения).
- **UI:** 
  - Слева: выбор сырья со склада.
  - По центру: интерактивная таблица состава (ввод процентов или граммов).
  - Справа: Инспектор смеси (прогноз расслоения, HLB баланс, итоговая стоимость).

### 3. `/synthesis` (Синтез-Навигатор)
- **Суть:** Проектирование химических реакций (как получить А из Б).
- **UI:** Node-based редактор (как в Unreal Engine Blueprints). Пользователь соединяет узлы-реактивы стрелками-реакциями, а система рассчитывает выход продукта и побочные вещества.

### 4. `/lab` (Анализ / Лаборатория)
- **Суть:** Глубокий анализ одной молекулы (то, что у нас есть сейчас).
- **UI:** 3D-просмотрщик, 2D-редактор, расчет квантовых и топологических свойств через WebGPU.

### 5. `/dev` (Управление Разработкой)
- **Суть:** Мета-раздел для управления проектом (уже реализован).
